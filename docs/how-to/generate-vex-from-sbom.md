# Generate VEX From An SBOM

Use this workflow when a repository already has a CycloneDX SBOM and you want
Vexcalibur to produce CycloneDX 1.6 VEX JSON during GitHub Actions.

The example uses local findings so it does not send package URLs, versions, or
SBOM-derived inventory to a public service.

## Prerequisites

- A CycloneDX JSON or XML SBOM committed to the repository, or generated by an
  earlier workflow step.
- A local Vexcalibur findings JSON file when using the no-network path.
- A trusted Vexcalibur Action ref and package spec.

For reviewed workflows, pin the action to a release ref or full release commit
SHA and use the package spec named in the
[compatibility reference](../reference/compatibility.md), such as
`vexcalibur==0.1.1`. Development workflows that use Git package specs, local
wheels, or source checkouts must set `allow-development-package-spec: "true"`.

## No-Network Workflow

This workflow checks out the repository, runs `vexcalibur generate` in offline
mode, verifies the output shape, and uploads the generated VEX JSON.

The example uses the current release pair from the compatibility table. Replace
`@v0.2.0` with the full release commit SHA when your organization requires
immutable action pinning.

```yaml
name: Generate VEX

on:
  workflow_dispatch:

permissions:
  contents: read

jobs:
  vex:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@9c091bb21b7c1c1d1991bb908d89e4e9dddfe3e0 # v7.0.0
        with:
          persist-credentials: false

      - name: Prepare output directory
        run: mkdir -p "$RUNNER_TEMP/vexcalibur"

      - name: Generate VEX
        uses: vexcalibur-dev/vexcalibur-action@v0.2.0
        with:
          package-spec: vexcalibur==0.1.1
          args: |
            generate
            sbom/cyclonedx.xml
            --offline
            --findings-file
            security/vexcalibur-findings.json
            --timestamp
            2026-06-23T00:00:00Z
            --output
            ${{ runner.temp }}/vexcalibur/cyclonedx-vex.json

      - name: Validate generated VEX
        run: |
          python - <<'PY'
          from pathlib import Path
          import json
          import os

          path = Path(os.environ["RUNNER_TEMP"]) / "vexcalibur" / "cyclonedx-vex.json"
          payload = json.loads(path.read_text(encoding="utf-8"))
          assert payload["bomFormat"] == "CycloneDX", payload
          assert payload["specVersion"] == "1.6", payload
          assert isinstance(payload.get("vulnerabilities", []), list), payload
          PY

      - name: Upload generated VEX
        uses: actions/upload-artifact@043fb46d1a93c77aae656e7c1c64a875d1fc6a0a # v7.0.1
        with:
          name: cyclonedx-vex
          path: ${{ runner.temp }}/vexcalibur/cyclonedx-vex.json
          if-no-files-found: error
```

Expected success signal: the workflow passes and uploads a `cyclonedx-vex`
artifact containing `cyclonedx-vex.json`.

For development testing against the latest Vexcalibur CLI source, use `@main`
with a Git package spec and explicitly allow development packages:

```yaml
- uses: vexcalibur-dev/vexcalibur-action@main
  with:
    package-spec: git+https://github.com/vexcalibur-dev/vexcalibur.git@main
    allow-development-package-spec: "true"
    args: --help
```

## Argument Handling

Each nonblank line under `args` becomes one command-line argument. The action
does not parse shell quotes or split on spaces. Put every command, option, and
option value on its own line when the value might contain spaces or option-like
text.

The `--output` path should point under `runner.temp` or another workflow-owned
directory. Upload the file with `actions/upload-artifact` when later jobs or
reviewers need the generated VEX document.

## Public OSV And Private Mirrors

Do not pass `--allow-public-osv` for private SBOMs unless sending package URLs
and versions to `https://api.osv.dev` is explicitly approved for the workflow.

For an internal OSV-compatible service, replace the offline source options with
`--osv-url`:

```yaml
args: |
  generate
  sbom/cyclonedx.xml
  --osv-url
  https://osv.internal.example
  --output
  ${{ runner.temp }}/vexcalibur/cyclonedx-vex.json
```

For intentionally public fixture data, use `--allow-public-osv`:

```yaml
args: |
  generate
  tests/fixtures/sbom/cyclonedx-json-simple.json
  --allow-public-osv
  --output
  ${{ runner.temp }}/vexcalibur/cyclonedx-vex.json
```

## CI Coverage In This Repository

The action repository validates this workflow shape in hosted CI by building a
Vexcalibur wheel from `vexcalibur-dev/vexcalibur@main`, generating VEX from an
XML SBOM with local findings, validating the JSON, and uploading the generated
artifact. See the [compatibility reference](../reference/compatibility.md) for
the current compatibility matrix.
